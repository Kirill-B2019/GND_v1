# Поддержка «много состояний» (слотов storage)

Реализована возможность чтения и записи нескольких слотов storage контрактов без выполнения байткода EVM.

## Чтение (view)

- **Слот по индексу в calldata:** при вызове view с `data` длиной 4+32 байта (селектор + один аргумент `uint256`) значение аргумента трактуется как номер слота. Возвращается 32-байтное значение этого слота из `contract_storage` (актуальное на последний блок).
- **Только селектор (4 байта):** возвращается слот 0 по умолчанию (обратная совместимость с `ganiToken()` и т.п.). Для селектора `owner()` (0x8da5cb5b) возвращается слот 1.
- Реализация: `core/state.go` — `CallStatic`, хелперы `slotKeyFromIndex`, `findSlotValue`, таблица `viewSelectorToSlot`.

## Запись (applyBlock)

При применении блока для транзакций типа `contract_call` по селектору (первые 4 байта `data`) выбирается обработчик. Поддерживаемые селекторы:

| Селектор   | Сигнатура           | Слот | Описание        |
|-----------|---------------------|------|-----------------|
| 0xc7d9116f | setGaniToken(address) | 0    | адрес GANI      |
| 0x13af4035 | setOwner(address)     | 1    | адрес владельца |

Реализация: `core/contract_call_result.go` — таблица `writeStorageSelectors`, функции `writeSetGaniToken`, `writeSetOwner`, `buildContractCallExecutionResult`.
