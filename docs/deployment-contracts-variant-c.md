# Деплой GND и GANI на контрактах (вариант C)

## Назначение

Документ описывает шаги перехода на модель «всё на контрактах» для GND и GANI (вариант C по ТЗ): деплой контрактов в текущую сеть без сброса, распределение начального supply по обновлённому ТЗ; данные в `native_balances` считаются устаревшими с фиксированного блока/времени.

## Обязательные шаги

1. **Деплой контрактов GND и GANI** в текущую сеть (отдельная транзакция/блок).
2. **В конструкторах:** выпуск на Treasury/мигратор по новому ТЗ (GND: начальная циркуляция на один адрес; GANI: вся эмиссия на один адрес).
3. **Скриптом или вручную:** с Treasury раздать по новому распределению (как в варианте A, но суммы — из нового ТЗ, а не из текущих `native_balances`).
4. **Зафиксировать в документации и, при необходимости, в софтфорке:** с такого-то блока/времени единственный источник истины по GND/GANI — контракты; данные в `native_balances` считаются устаревшими.

## Контракты

- **GNDToken.sol** — `tokens/standards/native/GNDToken.sol`  
  - Конструктор: `(initialSupply, treasuryAddress)`.  
  - Начальная эмиссия по ТЗ (например 100M * 10^18).  
  - Минтинг после деплоя отключён (`mint()` всегда `revert`).

- **GANIToken.sol** — `tokens/standards/native/GANIToken.sol`  
  - Конструктор: `(treasuryAddress)`.  
  - Фиксированная эмиссия 100M * 10^6 (6 decimals) в конструкторе на `treasuryAddress`.  
  - Минтинг отключён навсегда.

## Регистрация в ноде

После деплоя контрактов необходимо:

1. Записать в БД:
   - **contracts:** одна строка на контракт (address = адрес задеплоенного контракта, owner = treasury и т.д.).
   - **tokens:** одна строка на токен (contract_id = id контракта, symbol = GND или GANI, name, decimals, total_supply, circulating_supply по ТЗ).
2. Заполнить **token_balances** начальным распределением (как после раздачи с Treasury по новому ТЗ).
3. Указать в **config/native_contracts.json**:
   - `gnd_contract_address` — адрес задеплоенного GND-контракта;
   - `gani_contract_address` — адрес задеплоенного GANI-контракта;
   - при необходимости `fee_collector_address` для сборщика комиссий;
   - `gndself_address` — мультиподписной кошелёк платформы (GNDSelf); при **owner = gndself_address** комиссия за деплой токена не взимается.

После перезапуска нода переходит в режим «всё на контрактах»: балансы GND/GANI читаются и изменяются только через `token_balances` (по token_id, связанному с этими контрактами). Таблица `native_balances` для GND/GANI больше не используется для учёта.

## Комиссия за газ

Комиссия за газ списывается в GND через обновление балансов в `token_balances` для токена GND (операционное отображение состояния контракта GND). Альтернатива — вызов метода контракта GND (например `transferFrom`) при наличии соответствующей интеграции в VM.

## Миграция данных (опционально)

Если нужно перенести старые балансы из `native_balances` в контракты (вместо простого объявления их недействительными), используется отдельный скрипт: чтение `native_balances` по GND/GANI, проверка баланса мигратора на контракте, вызов `transfer` от имени мигратора на каждый адрес. После успешной миграции — пометить миграцию выполненной (флаг в БД/конфиге).

## Критерии приёмки (по ТЗ)

- GND и GANI существуют только как контракты; в коде ноды нет путей учёта их балансов в `native_balances`.
- Правила эмиссии (фикс. supply GANI, лимиты GND, отсутствие публичного mint у GANI) реализованы в Solidity.
- Определён и выполнен вариант C: новый деплой с новым распределением и документированием отмены старых native-балансов.
- Комиссия за газ списывается в GND через контракт/ token_balances.
- Документация и конфиги обновлены; адреса контрактов зафиксированы в `config/native_contracts.json`.
