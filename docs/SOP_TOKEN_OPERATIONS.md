# SOP: Операции с токенами и контроллером

Операционные процедуры для вызовов контрактов токенов и NativeTokensController. Для внешнего аудита и эксплуатации.

---

## 1. Кто и когда вызывает функции

### NativeTokensController

| Функция | Назначение | Кто вызывает | Когда / условия |
|--------|------------|---------------|------------------|
| `setGndToken(address)` | Установить адрес контракта GND (один раз) | owner (gndself_address) | После деплоя GNDToken; повторный вызов запрещён |
| `setGaniToken(address)` | Установить адрес контракта GANI (один раз) | owner | После деплоя GANIToken; повторный вызов запрещён |
| `mintGANI(to, amount)` | Выпустить GANI на адрес | owner | Доп. эмиссии в рамках лимита 100M; процедура и лимиты — внутренний регламент |
| `setKycGnd(user, status)` | Включить/выключить KYC для user на GND | owner | По результатам off-chain KYC-процедуры |
| `setKycGani(user, status)` | Включить/выключить KYC для user на GANI | owner | Аналогично |

### RWA-токен (GNDRWAToken)

| Функция | Назначение | Кто вызывает | Когда / условия |
|--------|------------|---------------|------------------|
| `setTransfersPaused(bool)` | Остановить/разрешить переводы | контроллер (от имени владельца логики RWA) | По инциденту или решению комплаенса |
| `setMintPaused(bool)` / `setBurnPaused(bool)` | Остановить минт/сжигание | контроллер | По инциденту или регламенту |
| `setFrozen(account, bool)` | Заморозить/разморозить адрес | контроллер | По решению комплаенса/суда |
| `setKycStatus(user, status)` | KYC по RWA-токену | контроллер | По off-chain процедуре |
| `snapshot()` / `setSnapshotBalance` / `setDividendsPerShare` | Снимок и параметры дивидендов | контроллер | По регламенту выплат; обеспечить соответствие балансам на дату |
| `mint` / `burn` | Эмиссия и сжигание | контроллер | По регламенту эмиссии RWA |

## 2. Ротация ключей owner

- **owner** контроллера задаётся при деплое из `config/native_contracts.json` (поле `gndself_address`) и в текущей реализации **не меняется** (immutable).
- **Ротация ключей EOA:** Если gndself_address — это адрес кошелька с одним приватным ключом, процедура ротации должна включать: генерацию нового ключа, сохранение нового адреса в конфиге для **следующих деплоев**, и при необходимости миграцию на новый контроллер (текущий контракт не поддерживает смену owner).
- Рекомендуется документировать: кто хранит ключи, как выполняется подписание транзакций (холодный/горячий кошелёк, мультисиг), периодичность смены ключей по внутреннему регламенту.

---

## 3. Смена контроллера

- В текущем коде **смены контроллера нет**: адрес контроллера в токенах GND, GANI и RWA задаётся в конструкторе и является immutable.
- При необходимости перехода на новый контроллер потребуется: деплой нового контроллера, при необходимости — деплой новых токенов с новым адресом контроллера и миграция ликвидности/учёта; обновление ссылок в интеграциях и конфигах. Конкретная процедура определяется регламентом проекта.
